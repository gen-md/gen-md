# Monorepo Documentation Skill

## Package README Structure

Every package README must include:

1. **Title** - Package name with badge
2. **Description** - One-line description
3. **Installation** - How to install
4. **Usage** - Quick start example
5. **API** - Key exports
6. **Related Packages** - Links to related packages in monorepo

## Cross-References

- Link to other packages using relative paths
- Reference shared types from @acme/core
- Point to root README for contribution guidelines

## Code Examples

- Use the package's actual exports
- Show integration with other monorepo packages
- Include TypeScript types

## Badges

Include these badges:
- npm version
- Build status
- License
