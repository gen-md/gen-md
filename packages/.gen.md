## Conventions

All package documentation should follow these patterns:

### Structure
1. **Header**: Package name with npm badge
2. **Description**: One-line description of the package purpose
3. **Installation**: Package-specific install command
4. **API/Features**: Document all public exports or features
5. **Examples**: Practical usage examples with code blocks
6. **Related**: Links to other gen-md packages

### Formatting
- Use TypeScript for code examples
- Include import statements in examples
- Use tables for command/API reference
- Keep explanations concise

### Package Naming
- npm scope: @gen-md/*
- Version: 0.1.0 (synchronized across monorepo)
- License: MIT

This configuration cascades to all README.gen.md files in package subdirectories.
