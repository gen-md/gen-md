## Overview
gen-md is a generative markdown framework using `.gen.md` configuration files.
It provides a consistent interface for developers to converse through their agents about their source code - .gen.md files become the ultimate spec for the codebase.

Key features:
- **Cascading configuration** - .gen.md files inherit from parent directories
- **Merge strategies** - Arrays: concatenate/dedupe/replace, Body: append/prepend/replace
- **Multi-platform** - CLI, VS Code, Chrome extension, Claude Code plugin
- **Conversational prompts** - Generate LLM-ready prompts with `--prompt` flag
- **Git context** - Infer context from local git history with `--git` flag
- **GitHub PR integration** - Multishot examples from merged PRs
- **PR output structure** - Generate PR-ready output with the `pr` command

## .gen.md File Format
```yaml
---
name: example
description: Description
context:
  - "./src/index.ts"
skills:
  - "./.agent/skills/skill.md"
output: README.md
---
Body content is the prompt.

One-shot examples use <example> blocks:
<example>
input specification
---
output result
</example>
```

## CLI Commands
All commands support `--prompt` (conversational output) and `--git` (include git context):

```bash
gen-md cascade <file> [--prompt] [--git]    # Preview cascade chain
gen-md validate <files...> [--prompt] [--git] # Validate .gen.md files
gen-md compact <files...> [--prompt] [--git]  # Merge multiple files
gen-md prompt <file> [--git] [--from-pr <n>]  # Generate prompt with examples
gen-md pr <file> [--from-pr <n>] [--create]   # Generate PR-ready output
```
